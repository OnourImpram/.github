# Contributing

Thank you for considering a contribution.

Most repositories under this account sit at the intersection of research software, agent safety, accountable AI memory, and human stakes workflows. Contributions are welcome when they improve correctness, safety, documentation, reproducibility, or usability.

## Principles

1. Keep changes small and reviewable.
2. Prefer evidence over assertion.
3. Do not add credentials, private data, clinical material, or unpublished personal information.
4. Add or update tests when behavior changes.
5. Document safety relevant behavior and failure modes.

## Pull Requests

Before opening a pull request:

1. Read the repository README and any project specific contributing guide.
2. Run the strongest available local checks.
3. Explain what changed, why it changed, and how it was verified.
4. Mark any remaining uncertainty clearly.

For agent, automation, career, clinical, or research workflow changes, include a short note on human oversight, data privacy, and reversibility.
